db.js IndexedDB
Promise-based CRUD wrapper for IndexedDB. Creates "RemindMeProDB" with "tasks" object store + 4 indexes.
ExportDescription

addTask(task)Insert via readwrite transaction → .add()
getAllTasks()Fetch all via readonly → .getAll()

deleteTask(id)Remove via readwrite → .delete()
notifications.js Engine
Multi-fire notification engine. Schedules N browser notifications at configurable intervals using setTimeout chains. Tracks active timers in a Map.
ExportDescription
requestPermission()Calls Notification.requestPermission(); returns boolean
showNotification(t,n)Creates Notification: "Reminder N of X", icon, tag, requireInteraction
scheduleMultiFire(t)Core: calculates delay→dueDate; sets setTimeout chain; checks ack flag before each fire; dispatches CustomEvent
cancelReminders(id)Clears all setTimeout handles from activeTimers Map
acknowledgeTask(id)Sets localStorage ack flag + cancels timers
snoozeTask(t, ms)Cancel → new dueDate = now+ms → snoozeCount++ → re-schedule
theme.js Theme
Dark/light mode manager. Reads/writes localStorage, sets data-theme attribute on <html>, listens for system prefers-color-scheme changes.
ExportDescription

toggleTheme()Flip dark↔light; return new value
initTheme()Apply on load + listen for system theme changes

1. Task Creation Flow

User taps [+] FAB
    │
    ▼
task-dialog.showModal()  ← defaults: date=now+1hr, priority=medium
    │
    ▼  user fills form (title*, date*, priority, category, count, interval)
form 'submit' event
    │
    ▼
Validate: title + date required → showToast("⚠️") if missing
    │
    ▼
generateId() → build task object (14 fields)
    │
    ▼
addTask(task) ──→ IndexedDB.tasks.add()
    │
    ▼
scheduleMultiFire(task) ──→ setTimeout chain registered in activeTimers Map
    │
    ▼
dialog.close() → showToast("✅ Created") → navigateTo('home') → renderHome()

2. Multi-Fire Notification Flow

scheduleMultiFire(task)
    │
    ▼
delay = new Date(dueDate).getTime() - Date.now()
    │
    ├── delay > 0 ──→ setTimeout(fireAll, delay)  [future task]
    └── delay ≤ 0 ──→ fireAll() immediately        [overdue/now]
              │
              ▼
        for i = 0 → reminderCount (or 999 if remindUntilDone):
            setTimeout(() => {
                │
                ▼
            Check: localStorage.get('task-ack-{id}')
                │
                ├── 'true'  → SKIP (user already tapped Done)
                └── 'false' →
                    ├── new Notification("Reminder {i+1} of {N}")  → Browser notification
                    └── dispatch CustomEvent('reminder-fired')     → In-app toast
            }, intervalMs × i)

4. Theme Switch Flow

User toggles switch in Settings
    │
    ▼
setTheme('dark' | 'light')
    │
    ├── localStorage.setItem('remindme-theme', theme)
    └── document.documentElement.dataset.theme = theme
              │
              ▼
    CSS rule [data-theme="dark"] { --bg: #1A1A2E; --text: #E8E8E8; ... } activates
              │
              ▼
    All var(--*) tokens swap instantly → entire UI re-themes (<1ms, no reload)

5. App Bootstrap Sequence (init)

init() — called on page load
    │
    1. initTheme()           → Apply saved/system theme
    2. openDB()              → Open IndexedDB connection
    3. setupNav()            → Wire 4 nav button click handlers
    4. setupTaskForm()       → Wire FAB + form + slider + checkbox
    5. navigateTo('home')    → Render home page (first paint)
    6. requestPermission()   → Ask for notification permission
    7. scheduleExisting()    → Load all pending tasks → schedule notifications
    8. setupReminders()      → Listen for reminder-fired events
    9. registerSW()          → Register Service Worker
       setupSW()             → Listen for SW messages (Done/Snooze)
   10. Connectivity listeners (online/offline)
       beforeunload → auto-backup
       startAutoBackup() if enabled
